Reliability Dashboard


About


This guide will help users to visualize and perceive assets through the eyes of IMS.

Scope

In this first release of IMS Reliability Dashboard , we enable users to visualize various categories of reliability as defined by IMS

Components

The basic idea of presenting this view is to ensure that all inmobi assets are under the radar of IMS, which eventually will act as the central source of truth & data store for all the assets.

  • Global IMS Pie View

    The pie view will present all the categories of reliability as defined by IMS along with contribution of assets (%) for each category. Further drill down is also possible for a selected category.

  • Global IMS Asset Table

    The Asset Table will present pre-defined attributes (i.e. Itag,Fqdn,Macid and IP) along wit the category of reliability tagged to it. This a global list of all the assets present in IMS. Further details can be fetched by clicking on the Itag URL.

Categories of Reliability

Various levels of reliability as defined by IMS is categorized on the following basis,

Criteria used for classification:

  • Are mandatory fields present?

  • Is latest version of IMS agent installed?

  • Is IMS agent active?

  • Physical or Virtual?

Based on the above criteria we have the following categories of reliability,

Category Description

Others Miscellaneous assets

How To's

On page load the complete pie view and global data table is presented without any category being selected by default. You can toggle between Global DataTable and Pie Table by selecting & unselecting a category respectively. To select a category do MouseClick on the desired item and to unselect again do MouseClick on the selected item.

Note: At present, you cannot toggle between pie category without unselection.

The varoius user interaction with the Pie View & DataTable is highlighted below.

  1. Interaction with Pie View

    • MouseOver on a Pie Category

      MouseOver on each pie to view the description of the selected category.

    • MouseClick on a Pie Category

      MouseClick on each pie to view the asset details of the selected category.

    • Export Options

      Click the options button in the top right corner of the pie view to export the chart as an image (PNG, JPEG) or PDF.

  2. Interaction with Table View

    • Search Assets

      Use the search bar in the table view to search for asset(s) based on preset filters (i.e. Itag, Fqdn, Macid and IP).

    • Filter Records

      Use the select box to filter number of records to be displayed per page (10 rows per page by default). Also click on the column header to sort the records based on the selected column field both in ascending &descending order.

    • Detailed Asset View

      Click on the Itag URL wherever visible to get greater details about the chosen asset.

How to make your asset reliable?

If you find your asset(s) in any of the unreliable categories, then the next steps to make it reliable are outlined,

  • if you are using puppet, check if it is broken or is there any other issue caused by it. Contact Us for help

  • If you are not using puppet, kindly follow the instructions given below

    1. Setup apt/yum repo

      • Configure apt/ yum repo locally

      • Issue the following commands to manually install ims agent,

        For Ubuntu,
        sudo apt-get update
        sudo apt-get upgrade
        sudo apt-get install ims-facter

        For CentOS,
        sudo yum update
        sudo yum install ims-facter

    2. Download & Install Packages

      • Download the following packages and dependencies

        Ubuntu Distro

          Dependencies
        1. facter
        2. daemontools
        3. daemontools-run
          4. Packages
        4. ims-nodejs
        5. ims-facter

        CentOS Distro

          Dependencies
        1. facter
        2. daemontools
        3. daemontools-run
          4. Packages
        4. ims-nodejs
        5. ims-facter

      • Manually install the downloaded packages with dependencies.

        Issue the following command to install the same 

        For Ubuntu,
        sudo dpkg -i *.deb

        For CentOS,
        sudo rpm -Uvh *.rpm
